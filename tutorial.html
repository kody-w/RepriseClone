üìπ Recording Demos

Method 1: Bookmarklet (Recommended)

Setup (One-Time)

  1. Open bookmarklet.html
  2. Make sure your bookmarks bar is visible:
    • Chrome/Edge: Cmd+Shift+B (Mac) or Ctrl+Shift+B (Windows)
    • Safari: View ‚Üí Show Favorites Bar
  3. Drag the "üìπ Record Page" button to your bookmarks bar

Recording a Page

  1. Navigate to the webpage you want to record (works on ANY site!)
  2. Click the "üìπ Record Page" bookmark
  3. A floating purple control panel appears in the top-right
  4. Click "Start Recording" (panel turns red)
  5. Interact with the page normally:
    • Click buttons and links
    • Fill out forms
    • Scroll the page
    • Type text
  6. Watch the event counter increase as you interact
  7. Click "Stop Recording" when done
  8. Click "Export Recording" to download the JSON file

üí° Recording Tips

  • The control panel is draggable - move it out of the way if needed
  • Works on sites that block iframes (Google Docs, Microsoft 365, etc.)
  • Each export gets a unique timestamp - you can make multiple recordings
  • Close the panel by clicking the √ó button

üé¨ Template Mode

Separate static content (headers, navigation) from dynamic content (chat messages, notifications).

What is Template Mode?

Concept

Template Mode lets you mark which parts of your page are:

  • Static: UI chrome, headers, navigation (stays the same)
  • Dynamic: Chat messages, notifications, data (changes/animates)

This separation lets you create reusable templates and edit dynamic content separately.

Creating a Template

Step-by-Step

  1. Load a recording in the player
  2. Click "üé¨ Template Mode" button
  3. A panel appears on the right
  4. Open "‚úèÔ∏è Edit Text" panel
  5. Browse the tree and find dynamic elements (e.g., chat messages)
  6. Select each dynamic element
  7. Click "üé¨ Mark as Dynamic"
  8. The element gets:
    • Blue background highlight in tree
    • üé¨ emoji badge
    • Blue element tag color
  9. Repeat for all dynamic elements
  10. Click "üì§ Export Template + Mapping"

What Gets Exported

template-{timestamp}.html

Full HTML page with dynamic elements MARKED (not removed). They stay in the template so you can edit them!

template-mapping-{timestamp}.json

JSON file listing all dynamic element selectors. Used by the animator to know what's editable.

Loading a Template

Auto-Detection

  1. Load a recording in the player
  2. Click "üé¨ Template Mode"
  3. Click "üì• Load Template"
  4. Upload a previously exported template HTML
  5. System compares template with current recording
  6. Auto-detects which elements are dynamic
  7. Marks them automatically!

üé® Static Page Animator

What is the Animator?

Purpose

The Static Page Animator is a specialized tool that:

  • Loads a static template HTML
  • Loads a mapping JSON (defines what's dynamic)
  • Lets you edit ONLY dynamic elements
  • Locks static elements (can't accidentally break the template)
  • Perfect for non-technical users updating content

üí° Tips & Tricks

Recording Best Practices

üí° Tips

  • Start fresh: Reload the page before recording for clean initial state
  • Slow down: Perform actions slightly slower than normal for better clarity
  • Test first: Do a test run to ensure everything records properly
  • Multiple takes: Don't worry about mistakes - just record again!
  • Move the panel: Drag the control panel out of the way before recording

Editing Efficiency

üí° Shortcuts

  • Element Picker: Fastest way to select elements - just click them!
  • Find Text: Use when you know the exact text you want to change
  • Pop-Out Editor: Use dual monitors? Pop out the editor for maximum space
  • Hide First: Hide unwanted elements before editing - cleaner workspace

Template Strategies

üí° Template Patterns

  • Chat demos: Mark messages as dynamic, keep UI static
  • Dashboard demos: Mark data cards as dynamic, keep layout static
  • Form demos: Mark input values as dynamic, keep form structure static
  • Notification demos: Mark notification content as dynamic, keep shell static

Troubleshooting

‚ö†Ô∏è Common Issues

  • Recording not working: Make sure you're using the bookmarklet, not iframe recorder for external sites
  • Player not loading: Check that your JSON file is valid (try opening in a text editor)
  • Elements not hiding: Hidden elements persist during playback due to rrweb DOM recreation - this is expected behavior
  • Snapshot comparison failing: Make sure you exported from the same recording you're comparing to
  • Dynamic elements not found: Re-export template from player - older templates may not have data attributes
