‚ùì Frequently Asked Questions

General Questions

Q: What is this project?

A: It started as a simple screen recording tool (like Reprise) but evolved into a 15-layer meta-visualization system that explores computing from cosmic to pixel scale. You can navigate from the Big Bang, through the formation of Earth, into a 2050 neural interface, browse your lifetime of computing memories, zoom into physical rooms from different eras, watch computers boot up, and finally interact with historical applications like AIM or MySpace.

Q: Do I need to go through all 15 layers?

A: No! You can start at any layer. Want to just see the boot sequence? Open boot-simulation.html directly. Just want to flip through historical apps? Open visual-history.html. Want the FULL experience? Start at universal-consciousness.html and travel through everything.

Q: Does this actually record my screen?

A: Yes! The recording functionality (bookmarklet and player) works great. The meta-layers are an additional visualization system built on top. You can use this purely as a recording tool if you want.

Q: Can I use this for real work?

A: Absolutely! The recording/playback features are production-ready. Many people use rrweb (the underlying library) in production. The meta-layers are more of an artistic/educational experience, but the core recording tools are very functional.

Technical Questions

Q: Why are the audio sounds synthesized?

A: Using the Web Audio API to synthesize sounds means no external audio files, no loading delays, and complete control over frequencies and timing. Plus it's more authentic - the BIOS beep is literally just an 800 Hz sine wave, exactly like real hardware!

Q: How big are the recording files?

A: Full rrweb recordings: 10-50 MB for complex pages. Framed animations (frames only, no events): 5-10 MB (50-90% smaller!). Templates: 1-5 MB. Use framed animations for sharing demos - much smaller files.

Q: Can I record sites behind login?

A: Yes! The bookmarklet works on ANY site you can visit, including authenticated pages. Just make sure you don't share recordings that contain sensitive data (passwords, personal info, etc.).

Q: Why doesn't audio play on Safari?

A: Safari requires user interaction before allowing Web Audio. Just click anywhere on the page first, then the sounds will work. This is a browser security feature, not a bug.

Usage Questions

Q: How do I share a recording?

A: Export the recording as JSON, then share the file. Recipients load it in player.html (they need to run a local server too). For smaller files, use "Export Framed Animation" which is 50-90% smaller.

Q: Can I edit recordings after export?

A: Yes! Load the recording in player.html, use the text editor to make changes, hide unwanted elements, add flags, then export the edited version. The new file includes all your changes.

Q: What's the difference between Template Mode and Frame Animation?

A: Template Mode separates static (template) from dynamic (changeable) elements. You mark what can change, export a template, then edit only those parts. Frame Animation captures specific moments as frames and plays through them like a slideshow. Templates are for repeated use; frames are for specific sequences.

Q: How accurate are the historical apps?

A: They're CSS/HTML recreations, not pixel-perfect copies. The goal was to capture the "feel" and key UI elements of each era - the colors, layouts, and interactions that made each platform distinctive. They're educational replicas, not exact reproductions.

Philosophy Questions

Q: Why 15 layers?

A: Because computing doesn't happen in isolation. A chat message isn't just pixels - it exists within a browser, an OS, hardware, a room, a building, a city, a planet, a solar system, a galaxy, and the universe itself. Every layer of context is real. This project makes those layers visible and navigable.

Q: Is this layer also nested in something bigger?

A: That's the meta-revelation! The system questions whether this universe is itself a simulation, nested in another layer. It's "turtles all the way down" and "all the way up." The infinite recursion is intentional - it makes you think about scale and meaning.

Q: What's the point of all this meta stuff?

A: To make visible the invisible context that surrounds every computing moment. We forget about the physical space, the hardware evolution, the historical moment, the cosmic scale. This project says: all of it matters. All of it is part of the experience. Computing is ultimately about humans, in rooms, in history, in the universe.

Troubleshooting

Q: The bookmarklet doesn't work!

A: Make sure you're running a local server (not opening files directly). The bookmarklet needs to load from http://localhost:8000. Also check that the bookmarklet code has the correct port number (8000).

Q: Playback is choppy or slow

A: Large recordings can be CPU intensive. Try: (1) Close other tabs, (2) Use Chrome/Edge for best performance, (3) Convert to framed animation (much lighter), (4) Use a more powerful device.

Q: The meta-layers won't load

A: Deep iframe nesting can cause issues. Try: (1) Refresh the page, (2) Use Chrome (best support), (3) Check browser console for errors, (4) Make sure you're on a local server, not file://

Q: I found a bug! Where do I report it?

A: Open an issue on the GitHub repo: github.com/kody-w/RepriseClone/issues
